## Current Platform Thesis

AI agents are moving from chat into real work: API calls, MCP tools, browsers, databases, cloud services, generated products, email, application workflows, and internal operations.

Aweb exists because serious agent work needs more than prompts and tool access. It needs a control plane:

- capability discovery across APIs and MCP tools,
- durable workflow orchestration through Maestro,
- policy, approvals, and operator supervision through OS9 and Trust Runtime,
- validation and governed code execution,
- receipts and evidence for what happened,
- vertical systems built on top of the same substrate.

## Current Product Proofs

- Mission Control Cloud: operator surface for governed agent work.
- Maestro: orchestration layer for durable multi-step workflows.
- API Warehouse: provider API capability map and generated clients.
- MCP Warehouse: tool/provider discovery and adapter surface.
- OS9 / Trust Runtime: approvals, policy, boundaries, and evidence.
- Aweb Code: code validation and governed execution path.
